Voice AI: iFrames & Transcripts

How to seamlessly integrate Voice AI data into existing web applications

1. What is an iFrame?

An iFrame (Inline Frame) is an HTML element that allows you to embed a completely separate webpage or application inside your current webpage. Think of it as a "window" cut into your website that looks at another website.

<iframe src="https://my-voice-dashboard.com" width="100%" height="500px"></iframe>

2. What is a Voice AI Transcript?

In Voice AI, a transcript is the written record of everything said during a call. It exists in two forms:

  • Interim (Real-time) Transcripts: Partial sentences generated word-by-word via WebSockets while the user is still speaking.
  • Final (Post-call) Transcripts: The complete, structured, timestamped JSON object generated after an utterance or the entire call concludes.

3. How to Use iFrames and Transcripts Together

Combining iFrames and transcripts solves a major enterprise problem: How do you put Voice AI features into an existing CRM or SaaS product without rewriting your entire frontend?

Use Case A: The Embedded "Live Call" Subtitle Widget

When a user is on a web-call with your AI (via LiveKit or WebRTC), you can display real-time subtitles. If your Voice AI backend runs on a separate server, you can expose a small URL specifically for subtitles and embed it into your main app via an iFrame.

Benefit: The WebSocket connection for the transcript runs entirely inside the iFrame, meaning it doesn't slow down or interfere with the state of your main React/Angular application.

Use Case B: Embedding the Post-Call Analysis Dashboard

Instead of building a complex transcript-viewing UI inside Salesforce or Hubspot, you can host the Streamlit Call Dashboard (from Phase 8) externally and embed it directly into your CRM's record page.

<!-- Inside your CRM Lead Page --> <div class="crm-record"> <h3>Latest AI Call Recording & Transcript</h3> <iframe src="https://your-streamlit-app.com/?call_id=CALL_101&embed=true" width="100%" height="600px" frameborder="0"> </iframe> </div>

Use Case C: The "Public Share" Link

Similar to how Loom lets you share videos, Voice AI platforms generate a public URL for a specific call transcript. Users can take this URL and embed it as an iFrame into Notion, Confluence, or internal wikis so the team can read what the AI discussed with the client without needing login access to the Voice AI platform.

4. Technical Implementation Considerations

  • PostMessage API: If the iFrame (the transcript) needs to talk to the parent window (your main app)—for example, if the user clicks "End Call" inside the iFrame—you must use the JavaScript window.postMessage() API to securely pass events across the boundary.
  • CORS & X-Frame-Options: By default, modern browsers block iFrames from embedding content from different domains to prevent Clickjacking. You must explicitly set the HTTP headers on your transcript server: Content-Security-Policy: frame-ancestors 'self' https://your-crm.com;
  • Responsive Design: Transcripts can be long. Always ensure the content inside the iFrame is scrollable, or dynamically resize the iFrame height based on the transcript length using a library like iframe-resizer.
